## Roadmap внедрения платформы

### Этап 1 — Базовый клиентский функционал (MVP)

- **Состав работ**:
  - Запуск `web` с разделами: главная, калькулятор, каталог, личный кабинет (каркас), заказы, загрузка моделей.
  - Запуск `api` с базовым GraphQL‑endpoint (`health`, каталог, заказы в минимальном объёме).
  - Локальное окружение (PostgreSQL, MongoDB, Redis, S3‑совместимое хранилище).
- **Критерии приёмки**:
  - Пользователь может зайти на сайт, просмотреть каталог и оформить простой заказ.
  - Работает расчёт стоимости через API (без глубокой оптимизации).
  - Есть базовая авторизация и личный кабинет с историей заказов (пусть пока пустой списком).

### Этап 2 — Админка и производство (MVP)

- **Состав работ**:
  - Разделы админки: заказы, производство, склад, оборудование.
  - Планировщик производства на основе модуля `planner.ts` и связка с БД.
  - Базовый складской учёт материалов и статусы оборудования.
- **Критерии приёмки**:
  - Оператор видит очередь заказов и может запускать их в производство.
  - Для каждого задания видно, на каком принтере и когда оно выполняется.
  - Видно остатки материалов и предупреждения о достижении минимальных уровней.

### Этап 3 — Финансы и внешние интеграции

- **Состав работ**:
  - Модуль финансов (выручка, расходы, прибыльность по заказам/периодам).
  - Интеграции с платёжными провайдерами и логистикой на основе адаптеров.
  - Начальная интеграция с выбранной CRM и одним маркетплейсом.
- **Критерии приёмки**:
  - Оплата заказов проходит через внешнего провайдера, статусы синхронизируются.
  - Есть базовый финансовый отчёт за период по заказам.
  - Клиенты и заказы экспортируются в CRM, заказы с маркетплейса импортируются.

### Этап 4 — Расширенная аналитика и AI

- **Состав работ**:
  - Сбор и хранение метрик в MongoDB (`production_metrics`, `analytics_snapshots`).
  - Модели/сервисы AI‑аналитики для оптимизации планирования и прогнозирования спроса.
  - Дашборды для менеджмента (загрузка оборудования, воронка продаж, прибыльность).
- **Критерии приёмки**:
  - Доступны отчёты по ключевым бизнес‑показателям в виде готовых дашбордов.
  - Планировщик учитывает исторические данные и даёт измеримое улучшение по SLA/загрузке.
